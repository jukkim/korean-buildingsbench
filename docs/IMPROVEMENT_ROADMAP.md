### C. 데이터 양/다양성 (합성 데이터 방향)

> **핵심 발견**: 문제는 "건물 수"가 아니라 **"건물당 반복 횟수"**
> - 현재(42K × 0.354ep) = **122회/건물**
> - BB (900K × 0.03ep) = **5.8회/건물**
> → 많은 건물을 아주 조금씩 보는 BB 방식 재현이 핵심
>
> **EXP-016 스윕 결과 (2026-03-16 확정)**:
> 2K=15.14% < 3K=15.16% < 4K=15.29% < 5K=15.55% < 6K=15.78% (단조 악화)
> → 2K가 스윕 내 최솟값. 1K 탐색 가치 있음 (U자 vs 단조 감소 확인)
>
> **⚠️ 샘플링 방법론 주의사항 (2026-03-16 확인)**:
> 현재 스윕은 각 스케일마다 **독립 랜덤 샘플링** (seed=42, 중첩 없음).
> - 1K / 2K / 3K / … 는 완전히 다른 건물 집합 → 스케일 효과와 샘플 선택 효과가 혼재
> - 2K↔3K 갭 **0.02%p**는 bootstrap CI(±~0.75%p) 내 → 노이즈로 처리
> - 4K→5K→6K 악화(0.2~0.3%p/step)는 반복 확인됨 → 실제 경향
> - **1K 결과 해석 기준**: 개선폭 ≥0.3%p면 유의미, 미만이면 샘플 우연 가능성
